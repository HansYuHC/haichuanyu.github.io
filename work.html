Schaeffler AG (2017年5月 - 2017年11月)

舍弗勒集团(Schaeffler AG)是一家专注驱动技术的科技公司, 公司业务涉及汽车、工业等领域，年营业额约130亿欧元(2016年)，全球员工超过85000人，总部位于德国巴伐利亚州黑尔措根奥拉赫市。

我于2017年5月份开始在舍弗勒公司进行了为期六个月的本科阶段实习。我所在的部门为链条传动的研发部门，实习目标为理解链条传动振动的原理并探究吸能器对于汽车发动机的凸轮轴以及曲轴减震的效果。

研究的目标主要是三个不同品牌车型的发动机的曲轴和凸轮轴，因为它们各自的传动原理以及曲轴和凸轮轴的按照位置各不相同，因此可以更全面地分析吸能器的工作原理和效果。

双质量飞轮可以有效降低发动机旋转的不均衡性而造成传动系的扭转振动。一是可以减少离合器在接合或分离时的冲击，另一点是可以减少发动机的震动。而我的实习目标即研究在双质量飞轮上安装吸能器(Tilger)，并通过MATLAB和模拟仿真软件，分析其对于减少链条传动系统振动的帮助。

项目中运用到了对旋转体不同频率转速所做的阶次分析，将旋转物体的运动分解成若干个不同固定频率的分运动的傅里叶分析，以及使用了舍弗勒公司内部的仿真模拟软件DyFaSim，来分析不同的汽车发动机安装吸能器后的减震效果。

最后将研究成果以PPT形式展示汇报。研究表明较低的阶次是振动的主要原因，并且吸能器在具有特定的阻尼度和共振频率时能产生较好的减震效果。加上吸能器的双质量飞轮与没有加的相比，可以更好地抑制发动机振动。

Stabilus SE (2021年11月 - 2022年7月)

斯泰必鲁斯(Stabilus SE)在全球拥有 6,000 多名员工和 16 个生产基地，是汽车和工业应用领域液压气动调节元件、液压减振器、机电驱动器和车门调节器的领先制造商。公司在其位于科布伦茨的总部将其自身设备和产品的开发和生产集中在一起。

我于2021年11月份开始在斯泰必鲁斯公司进行了为期八个月的研究生阶段自愿实习。我所在的部门为公司的研发创新部门 (SDI-Entwicklung Innovation)。 实习的主要内容为对于公司的创新项目"Silent Railway"的支持，即对于铁路列车，轨道及其接触系统的噪音和振动控制的研究。

"Silent Railway"项目的灵感来自于主动降噪耳机。如果把铁轨及其系统想象成一个整体，则该项目试图给铁轨系统带上一个主动降噪耳机，从而减少振动产生的噪音，使得周围的居民生活质量提高。课题的难点则在于主动降噪(ANC, Active Noise Cancellation)的绝大多数应用场景都是封闭或较小的区域，而对于铁轨这种开放式区域是否适用该技术，值得进一步研究。

开始项目后我先阅读了大量的相关文献，其中既包括铁路系统的各种标准，噪音及振动的相关技术文献，也有主动降噪技术的相关文章。之后便运用MATLAB和Simulink搭建仿真模型，嵌入各种不同的ANC模型的算法，并比较它们相互的优劣。同时我们还与ESI公司合作，利用他们的SimulationX多体模拟仿真测试软件，来测试不同算法的减振降噪效果。

中间的图片为ANC多种不同算法中的一种的Simulink模型。可行性报告显示，ANC的减振降噪概念可以运用到铁轨的仿真模型中并取得一定的效果，如果同时加入PID控制器则可以的达到更好的效果。

最后将研究成果以PPT形式展示汇报，并且生成了一份可行性报告。研究分析了各种不同的ANC算法应用在铁轨系统中的效果，并指出，需要找到合适的执行器来生成所需要的抑制振动的反向振动。并且由于条件所限，并没能进行对于铁轨的实物研究，只有仿真模拟结果。

Bertrandt AG (2022年7月 - 2024年3月)

Bertrandt AG 是一家位于巴登-符腾堡州埃宁根(Ehningen)的提供机械工程方面研发服务的上市公司。约有 14000 名员工在 50 多个地点工作，主要位于欧洲，但也在美国和中国，同时还为汽车和飞机行业的客户提供服务。

在卡尔斯鲁厄理工学院机械制造专业研究生毕业之后，我于2022年7月入职埃宁根的Betrandt AG，担任研发工程师。

在Bertrandt AG我在项目组"CarIT Security Testhouse"工作。项目主要内容是为主机厂梅赛德斯奔驰公司研发测试不同的车辆电子控制单元(ECU, Electronic Control Unit)。具体为在特制的测试台架上研发测试ECU的安全性能相关的功能，利用CANoe生成测试报告，分析报告中的错误，并且进行错误分析及追踪。

每周会有不同数量的ECU运送到我们项目组来进行测试。在ECU被接收到之后，首先会进行登记，我们会用到Jira以及Confluence来进行版本管理以及ECU的登记。之后会对ECU的软件版本和硬件版本进行读取，如果与预期的不一样，则需要将ECU与电脑连接，并通过DTS Monaco进行ECU的程序刷斜，以使其拥有正确的测试用软件及硬件版本。

需要注意的是不同的ECU拥有不同的总线通信系统CAN， Flexray或者是Ethernet。对于前两者会有High和Low两条线来用于通信，Ethernet则是有一个Wakeup Line。因此在ECU连接时需要注意总线的区分。

在ECU正确连接及刷写完毕之后，需要在CANoe中配置正确的测试环境。不同的ECU以及总线通信系统需要不同的测试环境。配置完成之后会简单检查一下ECU是否会对测试环境的询问做出应答，如果没有，则需要重新检查一遍

随后会在测试环境中首先进行ECU调试，这里会有大概20个测试用例，持续时间5到30分钟不等。如果调试过后生成的测试报告里没有严重的错误，则可以进行ECU的完整测试，大概200至500个测试用例，其时长为5到10个小时不等。

具体的测试用例的内容则会集中于ECU的安全性能。奔驰公司自己研发了ZenZefi系统，即中央证书管理系统(Zentrale Zertifikatsverwaltung)。每一个ECU都预先安装了证书,分别是Root(根证书)，Backend(后端证书)，Intermediate(中间证书)和ECU证书。如果没有一个完整的证书链，则该ECU无法进行安全测试。

安全变体编码，安全程序刷写以及安全机载通信，是主要的测试项目。除此之外还会进行TLS(Transport Layer Security, 传输层安全)测试，Gateway Test(网关测试)等一些其他测试。

在测试报告生成之后，我们会对其进行错误分析，并利用奔驰公司的STARC错误追踪登记系统进行报告上传，并与研发同事讨论分析错误原因，进行改进。

Mercedes-Benz AG (ANÜ von Bertrandt) (2024年4月 - 2025年9月)

梅赛德斯奔驰(Mercedes-Benz)是一家以豪华和高性能著称的德国汽车品牌，总部位于德国斯图加特，旗下产品有各式乘用车、中大型商用车辆。公司成立于1886年，年营业额超过1500亿欧元，全球范围有超过17万名员工。

在Bertrandt的"CarIT Security Testhouse"项目结束后，我被外派到位于Sindelfingen的梅赛德斯奔驰公司，就职于质量管理部门的增产工厂(Anlauffabrik)，从事车辆静态调试工程师的工作，协助研发原型车辆并且对于汽车电子控制单元在测试过程中出现的问题进行分析解决。

我所负责的项目为奔驰公司的223M系列，即对S级以及迈巴赫S级轿车进行静态调试。对于一个系列的车型，通常会根据研发阶段分为E，A和B阶段。E阶段即研发阶段，A和B则为正式量产之前的调试阶段，之后会进行到Pro1和Pro2正式生产阶段。每一个阶段都会有若干车辆运送到增产工厂以测试最新的软件版本，在包括静态调试，动态调试以及扩展功能测试在内的调试工作完成之后，车辆会被送到研发部门进行下一阶段测试。

静态调试用到了奔驰的IS-Tester，这是一个专门用于自动化测试的手提式便携测试仪。在与车载OBD接口连接之后，IS-Tester即可以开始对被测车辆进行不同的测试。测试完成后的报告会上传到奔驰的服务器以供分析及下载。测试脚本是基于C语言开发的NiSP(Neue Intelligente Schnittstelle Programm)语言。如果测试报告中有需要进一步分析的错误，则会对生成报告的脚本逐行检视，并联系研发部门同事分析解决。

在分析测试报告的错误并解决的同时，我也利用Python脚本对不同研发阶段的各个车辆的错误情况及错误处理时间进行了分析，以便优化流程，更高效地进行静态调试及其他步骤的工作。
